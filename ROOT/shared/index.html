The spine (same in every chamber)

00_CORE meta · 10_WHAT definition · 20_WHY purpose · 30_HOW operation · 40_WHO entities · 50_WHERE location · 60_WHEN planning · 70_HOWMUCH magnitude · 00_WORK work · 10_STATUS now+history.

Placement law: one canonical home + read-only facets; Entity ≠ Work ≠ Event. Full rules in 03_CORE_Rules.

How to navigate & manage this memory

  1. Enter a chamber (the three doors above) or jump straight to A_REFERENCE · B_OPERATIONS · C_OPERATIONAL_KIT. Each chamber index has a chamber switcher and lists every folder with a one-line summary.
  2. To find or place anything, follow the guide skill_use_structure.md and the routing rules in 03_CORE_Rules.
  3. Machine site-maps (for agents): A_REFERENCE/index.json, B_OPERATIONS/index.json; the full folder tree is _TREE.txt.
  4. Each agent has a private, scaled copy of this structure under agents/<id>/ that consolidates its periodic STATUS back into the shared memory.
