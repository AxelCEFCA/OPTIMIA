# 30_HOW - Operation [BASE]

> **5W2H - HOW:** How does it work? Playbooks, procedures, processes, methods, knowledge, integrations, policies (the definition, not the run).

## What goes here
- **Playbooks** (end-to-end) and **procedures** (step-by-step SOPs).
- **Processes / state machines** (the definition of a lifecycle).
- **Methods**, **knowledge base**, **integrations**, operating **policies**, output **templates**, **automations**.

## What does NOT go here
- The **running instances** (tickets, jobs) -> `00_WORK`.
- Memory-page templates -> `02_CORE_Templates` (vs business output templates `38_HOW_Templates`).
- Configuration of automations -> `06_CORE_Control`.

## How to read the derivatives (laws)
1. **Canonical home + facets (P1)**: each thing has ONE home here; its other dimensions are read-only `facet:` pointers in the frontmatter, never copies.
2. **One catalogue (P5)**: the 4-letter code namespace lives in `05_CORE_Glossary`; trees here are *projections* of it.
3. **Plans emit facts (P3)**: lifecycle events are pointed at `12_STATUS_Daily`, not transcribed here.
4. **Generated index (P7)**: `index.html` / `index.json` are generated, not hand-edited.

## Derivatives
- `31_HOW_Playbooks` - End-to-end scenarios (a full play).
- `32_HOW_Procedures` - Step-by-step SOPs.
- `33_HOW_Processes` - State machines / process definitions.
- `34_HOW_Methods` - Methodologies.
- `35_HOW_KnowledgeBase` - Reference knowledge that informs operation.
- `36_HOW_Integrations` - How to connect to external systems.
- `37_HOW_Policies` - Operating policies.
- `38_HOW_Templates` - Output / business templates.
- `39_HOW_Automations` - What the agents/automations do.

## Mark
[BASE] - base folder of the HOW family (5W2H spine).
